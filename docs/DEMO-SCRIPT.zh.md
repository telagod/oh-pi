# 2 分钟演示脚本（oh-pi + ant-colony）

> 目标：让第一次接触的人在 2 分钟内理解：
> 1) oh-pi 是一键配置入口；
> 2) 蚁群是可选的高阶能力；
> 3) 什么时候用蚁群更划算。

## 场景准备（录制前）
- 干净终端窗口
- 一个中等复杂度仓库（至少 3 个相关文件）
- 已安装 Node 20+

## 时间线

### 0:00 - 0:25 ｜ 一键配置
```bash
npx oh-pi
```
讲解词：
- 「先不用懂架构，先把 pi 配好。」
- 「oh-pi 的定位是入口：把 provider、主题、扩展一次性装好。」

### 0:25 - 0:45 ｜ 启动 pi
```bash
pi
```
讲解词：
- 「现在是可用状态，不做复杂操作也能立即开发。」

### 0:45 - 1:30 ｜ 触发蚁群执行复杂任务
示例目标：
- 「扫描仓库，新增一份重构计划文档，只改 docs 下一个文件，并复核输出质量。」

观察重点：
- `COLONY_SIGNAL:SCOUTING`
- （必要时）`COLONY_SIGNAL:PLANNING_RECOVERY`
- `COLONY_SIGNAL:WORKING`
- `COLONY_SIGNAL:TASK_DONE`
- `COLONY_SIGNAL:COMPLETE`

讲解词：
- 「复杂任务才启用蚁群，单文件修改不值得上蚁群。」
- 「如果 scout 输出不稳定，会进入恢复回路，不会直接崩。」

### 1:30 - 2:00 ｜ 结果与边界
讲解词：
- 「你拿到的是结果 + 过程信号，不是黑盒。」
- 「适合蚁群：多文件、可并行、需要审查。」
- 「不适合蚁群：单文件小修、低复杂度任务。」

## 结尾文案（可直接念）
「oh-pi 先解决‘能用’，蚁群再解决‘复杂任务效率’。先把入口体验做好，再逐步释放高级能力。」
