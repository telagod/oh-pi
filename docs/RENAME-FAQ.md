# Rename FAQ

## What is changing?

The project is moving away from the legacy/bootstrap name `oh-pi` toward:

- **Brand:** `Agent Colony`
- **Formal product name:** `Agent Colony for Pi`
- **Future canonical repo/package name:** `pi-agent-colony`

## Why rename at all?

Because the current `oh-pi` name is confusing in the wild and overlaps with another project using the same name.

## Is this a fork of the other `oh-pi`?

No.

They are different projects.

## Is `oh-pi` going away immediately?

No.

`oh-pi` is currently the transition/bootstrap/package name. The rename should happen with a compatibility window, not as a hard overnight break.

## What should users call the product now?

Use:

- **Agent Colony**
- or **Agent Colony for Pi**

## What will the canonical install name become?

Planned direction:

- `pi-agent-colony`

## Will existing users be broken?

That is not the goal.

The intended migration path is:
1. publish the new canonical package name
2. keep `oh-pi` as a compatibility bridge for a transition period
3. document the migration clearly

## What is the practical meaning right now?

Right now:
- the product should be talked about as **Agent Colony**
- the current npm/bootstrap name is still `oh-pi`
- the rename execution path is being prepared
